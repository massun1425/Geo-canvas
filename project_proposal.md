# アプリケーション企画書：旅行軌跡＆写真自動分類アプリ

## 1. アプリの目的・コンセプト
旅行中の移動ルートを地図上で可視化し、道中で撮影した写真を機械学習で自動分類して、ひとつの「旅行記」として美しく保存・振り返りができるWebアプリケーション。友人たちとの1週間の東北ドライブ旅行のような、移動距離が長く写真が多くなるイベントの記録に最適化する。

## 2. 主要機能（要件）
* **写真の自動マッピング機能（コア機能）**
    * スマートフォンの標準カメラ等で撮影した写真をアプリにアップロードする。
    * 画像データに含まれるExifデータ（緯度・経度・撮影日時）をバックエンドで抽出し、地図上の正しい位置にピンとして配置する。
* **移動ルートの可視化機能**
    * Google Maps API（または類似の地図API）を利用し、抽出した位置情報を時系列順に繋いで、移動した軌跡（ポリライン）を地図上に描画する。
* **機械学習による画像自動分類機能**
    * ニューラルネットワークを用いた画像認識モデルにより、アップロードされた写真を「風景」「集合写真（友人たちとの写真など）」「食事」などのカテゴリに自動分類してギャラリー表示する。
* **（拡張機能）アプリ内直接撮影**
    * ブラウザのAPIを利用し、Webアプリ上から直接カメラを起動。現在地のGPS情報と共にその場で写真を記録する機能（開発の後半で余裕があれば追加）。

## 3. 想定される技術スタック（開発環境）
実務を意識し、以下のモダンな構成で開発を進める。

* **開発環境:** macOS (Apple Silicon M1環境を想定), VS Code
* **バックエンド (API・処理):** Python (FastAPI または Flask)
    * 機械学習モデルの統合や、Exifデータの抽出処理を担う。
* **機械学習モデル:** TensorFlow または PyTorch
    * CNN（畳み込みニューラルネットワーク）を用いた画像分類モデルの構築。
* **データベース:** PostgreSQL
    * 写真のメタデータ（URL、緯度経度、撮影日時、分類結果）やユーザー情報の保存。時系列データや位置情報を効率よく処理するためのスキーマ設計が腕の見せ所。
* **フロントエンド (UI/画面):** JavaScript (React または Vue.js), HTML/CSS
    * ブラウザ上で地図を動かし、写真をアップロードする画面の構築。

## 4. 面接官に刺さる「アピールポイント」
* **実体験に基づく課題解決:** 「自分たちの旅行を便利に記録したい」という明確な動機があり、プロダクトへの熱量が伝わる。
* **データベースの最適化:** 単なるCRUD（作成・読み取り・更新・削除）だけでなく、時系列に沿った位置情報データをPostgreSQLでどう効率的に設計・処理したか（スキーマ設計の工夫）を語れる。
* **複合的な技術力:** 「Webアプリケーションの基礎」「外部API（Google Maps）の連携」「ニューラルネットワークを用いた画像処理」という3つの高度な技術を統合している点。
